## 10.4 테스트 주도 개발

## 10.5 제스트 사용하기

test 파일을 먼저 작성하고 코드를 짜고 테스트가 통과하는 걸 확인한다

```tsx
// function.js
export function order(items) {
  const total = items.reduce((price, item) => price + item.price, 0)
  return {
    orderItems: items,
    total,
  }
}

// function.test.js
import { order } from './function'

const menuItems = [
  {
    id: '1',
    name: 'Tatted Up Turkey Burger',
    price: 5,
  },
  {
    id: '2',
    name: 'Lobster Lollipops',
    price: 5,
  },
  {
    id: '3',
    name: 'Motley Quey plull porkt',
    price: 5,
  },
]

const result = {
  orderItems: menuItems,
  total: 15,
}

test('Build an order object', () => {
  expect(order(menuItems)).toEqual(result)
})
```

describe()
이 함수는 관련이 있는 여러 테스트를 하나로 묶어준다. 예를 들어, 비슷한 함수에 대한 몇 가지 테스트가 있다면, describe 문장으로 모든 테스트를 감쌀 수 있다.
그러면 테스트 러너가 테스트를 묶은 블록을 만들어준다. 이에 다라 터미널에 표시되는 출력 화면이 좀 더 조직적이고 읽기 쉬워 진다.

TDD 사이클 : 테스트를 먼저 작성하고, 그 테스트를 통과하기 위한 코드를 작성한다.

### 10.6 리액트 컴포넌트 테스트하기

리액트 컴포넌트를 Jest 를 이용해 테스트 방법을 공부했다
이를 이용해 리액트 DOM 엘리먼트들의 렌더링 결과들을 검증할 수 있다.
리액트를 초기화할 때 setupTest 파일을 이용해 jest expect extend 를 쉽게 사용할 수 있다
